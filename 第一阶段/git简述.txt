
git 和 github的区别 
git：是一款软件，这款软件是一个分布式仓库，主要的功能是存储，版本控制
github：是一个网站，他的主体是一个服务器，里面运行着git 这个网站基于git做了很多功能，比如记录你什么时候提交了代码等等功能

为啥这么如此重视git 必须完成？
      1.工作中的基础工具，所有人都会 你不会？
      2.有利于知识的积累，你积累三年代码你会知道他的好处的 
      3.向大神学习的最好方法，github上有无数的开源项目，牛逼的，好玩的，前沿的技术等等 多逛一逛
      4.看成一个提升效率的工具，如果你做什么东西 怕东西丢而且还想知道自己什么时间改了什么，回到历史版本等，git无疑是你最好的帮手，几行命令会帮你完成极其复杂的操作
      5.监督你们的进度，离的很远，没办法现场亲自带你们，我一直想办法消除距离的影响，不过这很难，因为大家时间也很宝贵，我希望尽量花最小的代价取得最大的价值。不希望变成一场浪费大家时间的闹剧

几个概念：

      java项目：用idea创建完项目 会生成一个叫项目名的文件夹 整个文件夹我称之为java项目，因为你整个文件夹拷贝到其他电脑，在用idea打开，idea会自动识别这是一个java项目，图标上和普通文件有明显的标识。
      
      本地仓库：git是个软件，安装完后 你需要告诉git 那个文件夹要交给git管理 例如你想把 test文件夹交给git帮你管理，那么你点击这个文件邮件，会有git的选项，打开命令行界面，执行 git init 命令，此时你的test文件就是一个本地仓库 名字叫test

       远程仓库：你的本地仓库别人是看不到的，如果你想把你的代码分享给别人，你需要一个公共的空间存储和你本地仓库一样的东西，然后别人去远程仓库拷贝你的代码，那么你上哪找这个公共的空间呢？
                              1.自己搭建git服务器 提供服务 你得会linux，买云服务器，买域名，    配置git等等
                              2.github 你在上面注册账号就可以免费使用git服务器 所有人都可以看见你的代码

任务：将java项目上传到远程仓库也就是github上

方法1：idea上新建java项目 选中java项目然后 在上面的工具栏点击VCS-> imort into version control -> share project on github
             首次点击后会弹出窗口 输入你的github账号和密码 然后一路点击即可
 
方法2： idea新建java项目，在github上新建一个仓库，复制github仓库网址 然后在idea点击terminal 打开命令行窗口
             按步骤执行以下命令：
             git add .
             git commit -m "首次提交"
             git remote add   origin github仓库网址   
             git push -u origin master

中间可能涉及问题：github账目设置问题，window密码凭据问题，安全验证权限等等 

说明：步骤少的要命，顺利做下来 几分钟就搞定，为啥给两周？
           因为会遇到让你各种抓狂的问题，莫名其妙的报错，还不知道因为什么，需要时间去百度搜索资料去解决，在解决的过程中逐渐熟悉git等操作

意义：动手解决问题的能力，谁也不是全能，不可能精通所有语言，解决所有问题，你眼里的大神，无非是经验丰富而已，所以你们要积累碰到问题，下次再碰到可以快速的解决，这样你也变成了别人眼里的大神。
